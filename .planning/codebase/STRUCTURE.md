# Codebase Structure

**Analysis Date:** 2024-03-24

## Directory Layout

```
python/mindflow_backend/
├── orchestrator/          # Execution loops, task flow definitions
├── services/              # In-memory and persistent state managers
│   └── orchestration/     # Specific services for tasks, plans, and todos
├── schemas/               # Pydantic models and data contracts
│   ├── orchestration/     # Orchestrator and Plan schemas
│   └── tools/             # Schema definitions for tools (e.g., Todo lists)
├── agents/                # LLM agent definitions, tools, and prompts
│   ├── prompts/           # Modular prompt segments
│   │   ├── core/          # Foundational personas
│   │   ├── specialized/   # Skill-specific instructions (e.g., planning)
│   │   └── composite/     # Pre-built combined prompts
│   └── tools/             # Exposed function calls for LLM usage
│       ├── orchestration/ # Tools for managing orchestrator states
│       └── planning/      # Tools for interacting with todo lists
```

## Directory Purposes

**`orchestrator/`:**
- Purpose: Orchestration control flow, routing logic, and execution loops.
- Contains: Flow decision scripts (`planning_flow.py`).
- Key files: `python/mindflow_backend/orchestrator/planning_flow.py`

**`services/orchestration/`:**
- Purpose: Runtime state managers handling data models and persistence.
- Contains: Services for plans, tasks, and todos.
- Key files: 
  - `python/mindflow_backend/services/orchestration/planning_service.py`
  - `python/mindflow_backend/services/orchestration/todo_planning_service.py`

**`agents/prompts/`:**
- Purpose: Modular storage for LLM system prompts.
- Contains: Python files exporting long-form Markdown strings defining agent behavior.
- Key files: 
  - `python/mindflow_backend/agents/prompts/core/orchestrator.py`
  - `python/mindflow_backend/agents/prompts/specialized/orchestrator_planning.py`
  - `python/mindflow_backend/agents/prompts/composite/full_orchestrator.py`

**`agents/tools/planning/` & `agents/tools/orchestration/`:**
- Purpose: Functions explicitly exposed to the Orchestrator/Planner LLM.
- Contains: Tool definitions mapping to backend services.
- Key files: 
  - `python/mindflow_backend/agents/tools/planning/todo_list.py` (Contains `WriteTodosTool`, `ReadTodosTool`, `FocusTodosTool`)
  - `python/mindflow_backend/agents/tools/orchestration/create_plan.py`

## Key File Locations

**Entry Points:**
- `python/mindflow_backend/orchestrator/planning_flow.py`: Determines if planning is required and orchestrates the execution loop for approved plans.

**Configuration:**
- `python/mindflow_backend/schemas/orchestration/planning.py`: Defines `PlanDocument`, `PlanTask`, `PlanStatus`.
- `python/mindflow_backend/schemas/tools/planning.py`: Defines `TodoItemContract` and schemas for tool injection.

**Core Logic:**
- `python/mindflow_backend/services/orchestration/planning_service.py`: Converts LLM intentions into `.md` plan files and processes human confirmations.
- `python/mindflow_backend/services/orchestration/todo_planning_service.py`: Normalizes plan complexity and tracks granular task progress in session memory.

## Naming Conventions

**Files:**
- Snake Case Services: `todo_planning_service.py`, `planning_service.py`
- Prompt Segments: Describe capability (`orchestrator_planning.py`) or base persona (`orchestrator.py`)

**Data Models (Schemas):**
- PascalCase with "Contract" suffix for external representations: `TodoItemContract`
- PascalCase for internal documents: `PlanDocument`

## Where to Add New Code

**New Planning Tool:**
- Implementation: `python/mindflow_backend/agents/tools/planning/`
- Schema Registration: `python/mindflow_backend/schemas/tools/planning.py`

**Modifying Prompts:**
- Core behavior changes: `python/mindflow_backend/agents/prompts/core/`
- New capability instructions: `python/mindflow_backend/agents/prompts/specialized/`
- Merging behavior: Update builders in `python/mindflow_backend/agents/prompts/composite/`

**New Plan Execution Logic:**
- Implementation: Add to or modify loops in `python/mindflow_backend/orchestrator/planning_flow.py`

## Special Directories

**`.plans/` (Temporary/Runtime):**
- Purpose: Generated by `PlanningService` to store Markdown summaries of `PlanDocument` structures for human review and logging.
- Generated: Yes
- Committed: No (Resides in tempdir or configured working path)
